# H ARCHIVES — Секретный архив организации Н

Атмосферная база данных рассекреченных материалов, аномалий, операций, инцидентов и архивных документов организации «Н».

## Возможности

- **Разделы архива**: Документы, Фотоархив, Досье, Аномалии, Инциденты, Операции, Персонал, Хронология
- **Система авторизации**: Firebase Authentication (email/пароль + Google)
- **Роли**: Гостевой режим (просмотр) и Организатор (полное управление)
- **Терминал доступа**: Командный интерфейс с поиском, навигацией и статистикой
- **5 уровней допуска**: Открытый, Ограниченный, Секретный, Совершенно секретный, Особой важности
- **Карточки объектов**: Код, уровень допуска, статус, описание, вложения, история изменений
- **Полнотекстовый поиск** по всем разделам
- **Панель управления**: Создание, редактирование, удаление записей, загрузка файлов
- **Визуальные эффекты**: CRT, скан-линии, глитч, VHS-помехи, шум экрана, мерцание
- **Адаптивный дизайн**: Телефоны, планшеты, ноутбуки, ПК

## Технологии

- Vanilla JS (ES Modules)
- Firebase (Auth, Firestore, Storage)
- CSS3 (анимации, глитч-эффекты, CRT)
- PWA (Service Worker, Manifest)

## Запуск

Открыть `index.html` в браузере или задеплоить на GitHub Pages / Vercel.

```bash
# Локальный сервер
npx serve .
```

## Деплой

- **Vercel**: Залить репозиторий, автоматически задеплоится через `vercel.json`
- **GitHub Pages**: Включить в настройках репозитория

## Firebase

Проект использует Firebase (`izunax-c1707`):
- Authentication: email/пароль + Google
- Firestore: коллекции `archive_documents`, `archive_photos`, `archive_dossiers`, `archive_anomalies`, `archive_incidents`, `archive_operations`, `archive_personnel`, `archive_timeline`, `users`
- Storage: файлы вложений

## Демо-доступ

Зарегистрируйтесь через форму или войдите через Google.
Первый пользователь с email из списка админов получает роль организатора.
